方法

方法是作用再接收者上的函数，接收者可以是除了接口和指针类型的任意自定义类型，所谓自定义，是不能直接把int这种给方法，但是给int 取一个别名，就可以赋予方法了。

最后接收者不能是一个指针类型，但是它可以是任何其他允许类型的指针

这段文字的意思是，在 Go 语言中，定义方法时，接收者的类型不能是一个指针类型，比如 *int 或 *string。但是，接收者的类型可以是一个指向其他类型的指针，比如 *Vertex 或 *Connection。这样的接收者可以修改它所指向的值，也可以避免在每次方法调用时复制值。如果接收者的类型是一个值类型，比如 int 或 string，那么方法只能操作接收者的副本，不能改变原来的值。

在go中，类型的方法和类型不必放在一起，只需在一个同一个包、

方法没有重载，但是允许不同类型有同名方法

方法的定义：func (类型)name(参数列表)(返回值列表){

} //注意这里的类型可以是指针类型和普通类型，指针可以改变数据，普通则只能得到一个copy

未导出字段：导入其他包的结构体只是导入一个类型，字段没有导出，必须通过方法导出，类似于Cpp的保护

包管理

go version 看版本

go的包都在go - pkg - mod 目录下面

go mod

go mod init path,初始化一个包管理

go.mod 提供了module, require、replace和exclude 四个命令

  • module 语句指定包的名字（路径）,module 路径不要引号
  • require 语句指定的依赖项模块
  • replace 语句可以替换依赖项模块，replace …… => ……
  • exclude 语句可以忽略依赖项模块

go get -u 升级所有模块 go list -m -u all 检查所有可以升级的package

[go工作区](官方博文：Go 1.18 工作区模式最佳实践 - 知乎 (zhihu.com))

go work init 后可跟go.mod本地目录，如果有多个go.mod 则用空格分开，注意，如果指定了go.mod，这些都会归工作区管，如果没，则go work init 会新建一个空工作区

添加go.mod go use -r 目录 ,其中，-r可选参数，如果有则遍历后面目录下面所有的文件的mod并添加，如果手动删除某个mod文件，只需再次go work use 。。。即可
