数据结构

新手村

基本术语和概念

数据：数值/非数值

数据元素： 组成数据的有一定意义的基本单位

数据项： 数据不可分割的最小单位

数据对象：数据的子集，性质相同的数据元素的集合

数据结构： 相互之间存在的一种或多种特定关系的数据元素的集合

逻辑结构&物理结构

逻辑结构： 数据对象中数据元素的相互关系，分为四种

- 集合结构
- 线性结构
- 树形结构
- 图形结构

物理结构： 逻辑结构在计算机内存储形式

  • 顺序存储结构
  • 链式存储结构

抽象数据类型

数据类型： 一组性质相同的值的集合及定义在此集合的操作的总称

抽象数据类型（Abstract data type|ADT）：一个数学模型及定义在该模型的操作

算法

算法： 解决问题的特定步骤，计算机指令的有限序列

算法效率的度量：

  • 时间复杂度 T(n) = O(f(n))

  • 空间复杂度 S(n) = O(f(n))

线性表

List: 零个或多个数据元素的有序序列

线性表的顺序存储

一段地址的连续存储单元依次存放线性表的数据元素

线性表顺序存储优缺点：

优点：

  • 快速存取元素
  • 插入和删除需要移动大量元素
  • 空间碎片
  • 容量难定

线性表的链式存储

为了表示每个数据元素和其直接后继的关系，每个数据元素都需存储自身信息【数据域】和后继位置【指针域】，指针域中存储的信息称为指针或者链。这两部分组成a元素的存储映像，称为结点（node）

链表的第一个结点的存储位置称为头指针，链表的最后一个结点的指针为NLLL，为了方便操作，在单链表第一个结点钱附设一个结点，头结点，数据域不太重要，指针域存储指向第一个结点的指针。

树和二叉树

基本概念

树

  • 节点深度： 从根节点到x结点的路径长度,根节点1

  • 节点层次： 根节点第一层，以此类推

  • 度： 节点的子树数目为节点的度，树的度为节点度的最大值

二叉树

基本信息

  • 节点度不超过二
  • 只有一个根节点
  • 左右子树具有次序，不可颠倒：左孩子，右兄弟

二叉树性质

  • 第 i 层至多z^(i-1)节点 ： 由于二叉树度不超过二，节点最多就是度全满为2，a1 = 1, q = 2, an = 1 * 2 ^(n-1)

  • 深度为 k 的二叉树至多2k-1个节点： 深度为k , 满层节点数目为: 1*(1-2^(k))/1-2 = 2^k -1

  • 对于任意二叉树，终端节点个数为m, 度为2的节点个数为n,那么m = n + 1 : 每增加一个度为2的节点，必增加一个度为0的终端节点，原因是增加的两个中有一个是增加的二原来占据的，初始状态n=1, m = 2,n = (1 + k), m = (2 + k), 故度为0的节点必度为2的节点多1个，二叉树皆是如此

  • 满二叉树 ： 每一层节点数都为该层最大数

  • 完全二叉树 ： 从上到下，从左到右，标号，和满二叉树的每个节点一一对应

    • 满二叉树的性质 ： 具有n个节点的完全二叉树的深度为： 假设深度为k, 则 2^(k-1)-1< n < =2^(k) -1 <=> 2^(k-1) < n + 1 <= 2^(k) <==> k-1 < log2(n+1) <= k <==> k = log2(n+1)向上取整。
    • 暂略

二叉树的存储

  1. 顺序存储

    从根节点开始，顺序存储，将完全二叉树的节点i定义在数组i-1处，非完全二叉树则和完全二叉树对照，存储在相应的分量中。

  2. 链式存储

    1. 二叉链表和三叉链表： 三叉链表多了指向双亲的指针域

    2. 线索二叉树： 有n个节点的二叉链表必然有n+1个空链域： 一个节点2个空链域，增加一个节点也增加一个空链域

二叉树遍历

  1. 先序后序中序遍历

    1. 根据遍历序列确定二叉树

      • 知道前中，中后，均可以确定二叉树： 如果只知道一个，无法确定根节点。知道两个，可以递归的知根节点，简单说就是一个如何分界的问题，先序遍历后序遍历第一个和最后一个元素为根，和中序搭配就可以分开左右子树，如果先后搭配，则无法分开左右子树

    2. 二叉树遍历算法的应用

      1. 建立二叉链表
